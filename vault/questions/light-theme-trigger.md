# Light theme — what triggers V2 implementation?

## Status

**Open** — awaiting team decision.

## Why it matters

Consilium is dark-mode-only through V1. Light theme is on the V2 list. Customer-driven theming requests are common in enterprise software; a defined trigger prevents the team from chasing one-off requests vs waiting for a real signal. (see [[consilium-design-guidelines]] §2.1)

## What we know so far

- Dark mode primary.
- Light mode is V2.
- No trigger condition named.

## What would resolve it

- Pick a trigger (e.g. "ship light theme when ≥3 paying customers explicitly request it" or "ship when a V2 accessibility audit requires it for low-vision users").
- Confirm the design tokens are already sufficient to support a light variant (the GitHub-classic-light palette is well-documented).

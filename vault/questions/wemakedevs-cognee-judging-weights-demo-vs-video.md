# WeMakeDevs Cognee Hackathon judging — what is the weight between live demo and submitted video?

## Status

**Open** — awaiting team decision.

## Why it matters

The 4-day-build / 2-day-pitch split is justified by the assertion that [[wemakedevs-cognee-hackathon|WeMakeDevs Cognee Hackathon]] is "async submission with video and slides." If video carries dominant weight, the 2-day pitch window may need to grow; if live demo URL gets dominant weight, the 2-day pitch window may be too generous. (see [[hackathon-judging-fit]], [[consilium-context-dump]] §10)

## What we know so far

- Submission requires: title, descriptions, tags, cover image, video, slide deck, GitHub repo, demo platform, application URL.
- Judging format implied to be async.
- No explicit weight published in the docs.

## What would resolve it

- Check the WeMakeDevs Cognee Hackathon's published judging rubric for the AI & Big Data Expo hackathon.
- If unclear: rehearse for both axes — a strong video that stands alone, plus a deployed demo URL that judges can hit.
